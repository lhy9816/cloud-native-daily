### 1. 📦 Kubernetes v1.36 工作负载感知调度大升级：Workload/PodGroup API 解耦 + 拓扑感知调度首秀

**原文标题:** Kubernetes v1.36: Advancing Workload Scheduling
**来源:** Kubernetes Blog | 2026-05-13（v1.36 周期核心特性，本周社区持续讨论）
**链接:** https://kubernetes.io/blog/2026/05/13/kubernetes-v1-36-advancing-workload-aware-scheduling/

🎯 **一句话精髓**: v1.36 将 Workload API（静态模板）与全新的 PodGroup API（运行时状态）彻底解耦，并首次落地拓扑感知调度与工作负载感知抢占，让 AI/ML 与批处理作业的原子化调度成为原生能力。

🔧 **技术栈**: kube-scheduler / scheduling.k8s.io/v1alpha2 / PodGroup scheduling cycle / Dynamic Resource Allocation (DRA) / Job controller

📐 **架构/方法亮点**:
- Workload API 退化为「静态模板」，运行时状态全部移交新 PodGroup API——职责分离让调度器可以原子化处理整个工作负载
- 新增 PodGroup scheduling cycle，实现「要么全调、要么不调」的原子语义（gang scheduling 的正式基础）
- 首批拓扑感知调度（topology-aware scheduling）+ 工作负载感知抢占（workload-aware preemption）进入迭代；ResourceClaim 支持 PodGroup，打通 DRA 动态资源分配

💡 **关键数据**: PodGroup API 属 scheduling.k8s.io/v1alpha2，完全替换 v1alpha1；Job controller 与新 API 的第一阶段集成已交付

🔗 **樱桃的调度视角**: 这条对 Master 是「核心中的核心」——拓扑感知调度直接关系到跨机架/跨可用区的 GPU 亲和性，PodGroup + DRA 则是让 KubeRay / Volcano 这类 AI 推理编排器未来能收敛回原 kube-scheduler 的关键一步喵～
